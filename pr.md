
# PG 심사용 결제 서비스 샘플 페이지 (PR용)

본 문서는 **PG 심사 목적**으로 실제 결제 플로우가 어떻게 동작하는지 명확히 보여주기 위한 **결제 페이지 샘플 기획서**입니다.
향후 **모바일 앱 전환**을 고려하여 **모바일 중심 UI (최대 너비 770px, 화면 중앙 고정)** 기준으로 설계합니다.

에이전트(프론트엔드/퍼블리셔)에게 전달하여 **PR 페이지 코딩**을 의뢰하는 것을 목적으로 합니다.

---

## 1. 페이지 목적

* PG사 심사를 위해 **실제 결제 상품, 지점, 금액, 사업자 정보가 명확히 노출**되는 페이지
* 사용자가

  1. 지점 선택
  2. 상품 선택
  3. 성함/전화번호 입력
  4. 결제 수단 선택
  5. 결제 요청
     을 진행하는 **실제 결제 UX 흐름**을 확인할 수 있어야 함

---

## 2. 레이아웃 기본 가이드

### 화면 규격

* **Max-width:** 770px
* **정렬:** 가로 중앙 정렬
* **배경:** #F7F7F7 (연한 그레이)
* **콘텐츠 영역:** 흰색 카드 형태 (radius 12px)

```css
.container {
  max-width: 770px;
  margin: 0 auto;
  background: #ffffff;
}
```

> 📱 모바일 앱 전환 대비: 전체 구조는 단일 컬럼, 스크롤 기반

## 8. 결제 수단 선택

> PG 심사 필수 노출 영역

### 예시

* 신용/체크카드
* 간편결제 (카카오페이, 네이버페이 등 – 실제 연동 예정 표시)

```text
(●) 신용/체크카드
( ) 간편결제
```

> ⚠️ 실제 PG 연동 전이라면
> "※ 본 페이지는 PG 심사용 결제 샘플입니다" 문구 하단 표시
